# AI 班务与日程管家

面向大学班长场景的智能任务管理 Web 应用。它不是普通待办清单，而是一个会理解通知、识别冲突、辅助决策、帮助重排全局秩序的班务管家。

## 这个项目在解决什么

班长日常常见的 9 个痛点：

1. 任务来源分散：通知、群聊、临时口头安排都容易漏。
2. 时间信息不完整：有的只有截止时间，没有开始时间和耗时。
3. 通知太长太绕：需要先“翻译成人话”才能执行。
4. 多件事同时撞车：不知道该先保哪一件。
5. 优先级不清：党团、班务、兼职、个人事务混在一起难排。
6. 紧急程度模糊：离 deadline 多近才算真的急，常靠感觉。
7. 关联对象复杂：导员、团支书、同学、老师牵一发动全身。
8. 插入突发任务后，全局顺序容易乱。
9. 事情不是一个动作，而是一串流程，容易做一半忘一半。

本程序对应的解决方式：

- `AI 智能录入`：把原始通知自动提取成结构化任务。
- `大白话摘要`：把公告内容压缩成一句能立即执行的话。
- `优先级引擎`：把党团、班务、校外、个人、单线任务放进统一排序体系。
- `冲突决策面板`：发生撞车时给出“保留两者 / 以新换旧 / 暂缓新任务”。
- `关系人名录`：按“导员 / 团支书 / 同学”等维度回看任务影响面。
- `子任务流程`：把“收表、汇总、上报”拆成可勾选步骤。
- `AI 一键重排`：把所有任务交给 AI 重新理顺时间秩序。

## 当前功能

- Next.js App Router + TypeScript + Tailwind CSS
- Zustand + LocalStorage 本地状态持久化
- SiliconFlow / DeepSeek-V3 智能解析接口
- 任务看板：分类视图 / 优先级视图
- AI 智能录入
- 冲突检测与冲突决策对话框
- 待办池机制
- 关系人名录与关联任务高亮
- 子任务勾选完成
- AI 全局自动重排预览
- 压力测试场景注入

## 启动方式

### 1. 安装依赖

```bash
npm install
```

### 2. 配置 `.env.local`

在项目根目录新建 `.env.local`：

```env
SILICONFLOW_API_KEY=你的真实密钥
SILICONFLOW_MODEL=deepseek-ai/DeepSeek-V3
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1/chat/completions
```

说明：

- `.env.local` 已在 `.gitignore` 中忽略，不会自动提交到 GitHub。
- 如果没有配置 `SILICONFLOW_API_KEY`，AI 解析和 AI 重排接口会直接返回错误提示。
- 推荐每次修改环境变量后重新启动开发服务。

### 3. 启动开发环境

```bash
npm run dev
```

打开：

```text
http://localhost:3000
```

### 4. 生产构建检查

```bash
npm run typecheck
npm run build
```

## 核心优先级算法

### 1. 绝对优先级

系统严格遵守以下顺序：

1. 党团事务
2. 班务
3. 非校内事务
4. 自我个人事务
5. 单线任务

在代码中，任务会被映射为 `P1 ~ P5`：

- `P1`：党团事务
- `P2`：班务
- `P3`：非校内事务
- `P4`：个人或私下事务
- `P5`：单线任务

### 2. 紧急程度

系统根据当前时间与 `deadline` 的距离打标：

- `24h 内`：紧急
- `72h 内`：普通
- `72h 外`：宽裕

### 3. 综合排序分

看板中的“综合分”会叠加以下信息：

- 绝对优先级
- 截止时间临近程度
- 预计耗时
- 关联对象数量与重要度
- 是否属于单线任务

### 4. 冲突处理原则

当两个任务时间重叠时，系统会比较：

- 优先级
- 时长
- 关联对象数量
- 截止时间

然后由用户在冲突决策面板中选择：

- 保留两者
- 以新换旧
- 暂缓新任务

### 5. AI 全局重排原则

点击“AI 智能理清秩序”后，系统会把全部任务交给 AI，并要求它：

- 严格遵守优先级顺序
- 先保障高优先级与近期限任务
- 尽量减少时间重叠
- 必要时把低优先级任务留在待办池

## 压力测试

项目内置了一组冲突测试场景：

- 周五下午 4 点塔斯汀兼职
- 导员临时通知党团会议

点击页面中的“注入冲突测试场景”即可验证：

- `P1 党团事务` 是否会压过 `P5 单线兼职`
- 冲突决策面板是否会正确出现
- AI 重排是否能把低优先级任务延后或转入待办池
